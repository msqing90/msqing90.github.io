Nodejs初探之一：用Express搭建项目－Step1

踏上程序媛这条路已经快3个年头了，写博客还是第一次。做这个项目呢，也是一个很偶然的机会。作为一个只会js的我，之前看到用node竟然能写后端程序，顿时为身为前端工程师倍感骄傲，哈哈，会js也能成为Fullstack Developer啦！所以，本人就用我的第一个node项目作为自己的开山之作。

好啦，咱们切入正题，此篇文章将会介绍我用Express做的第一个‘求职者一项填表系统’的后台部分，首先来说说我为啥会选node来做这个项目：

  • 用一种语言和一种编程风格来解决两个端（Client和Server）的开发工作。
  • 和传统的后端语言相比，因为node是基于Chrome的Javascript引擎V8，它天生就具有非阻塞，高性能的优点。
  • 基于node的Express非常流行、轻量级、简单易用，具有丰富的中间件，比如：在后几篇文章会讲到的解决处理前后端跨域请求的CORS等。

下面这幅图介绍的MEAN STACK(Mongodb＋Express＋Nodejs)从客户端到服务器端的通信图：

在实际动手Express开发之前，我首先关心这么几个问题：

  1. 怎样配置开发环境，包括Nodejs，Mongodb，Express，怎样让应用跑起来。
  2. 客户端发给服务器端的不同URL定义的API，Express是如何用自己的业务逻辑进行匹配。
  3. Express如何连接数据库。
  4. Javascript代码和客户端环境下有什么差异，例如回调的实现是否和客户端一样。

这篇文章只涉及到Express部分，Mongodb和Nodejs在后续文章中会讲到。

下面，来给大家讲一下我是怎样一步步搭建这个项目的。

配置和运行：

我在这里记录两种配置和运行方法：

  • 一种是开发Node应用最常用的方法，也是标准方法。
  • 一种是用Express自己提供的工具创建的方法，相当于快速创建scaffolding项目。

标准方法：

首先下载安装Nodejs，官网在nodejs.org。需要说明的是安装好Nodejs，也包含了一个重要的命令行工具NPM(Node package Management)，用来管理Nodejs所需要的包。在Nodejs上开发的任何应用都是一个package。Express就是Nodejs上用来开发Web应用的一个package，我们要开发的web应用其实也是一个package。

所以安装好Nodejs后，我们首选就是要用NPM来创建我们自己的Package，然后再用NPM安装Express。

创建package.json

npm init会提示你输入很多选项，这些选项最后都被用来创建了一个package.json文件，用来描述当前这个package的所有配置信息。关于package的更多内容可以参考官方规范：https://docs.npmjs.com/files/package.json。
其他问题，直接回车选择默认值就行。下面这个问题，需要你输入一个Javascript文件名，作为要开发的应用的入口文件。也就是类似Java应用中包含Main方法的那个类。

使用Express Generator创建方法：

首选安装Express Generator。没错Express Generator也是一个Node的package，所以有了上面的安装经验，就知道如何安装了。只是这次加一个参数-g

-g或者-global的作用是让当前package成为全局的，而不同于之前安装Express那样只是把包安装到某一个指定的目录里。全局的Package可以在任何目录下使用，所以方便将来创建其他Express项目。
